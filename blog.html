Vulnerability Overview

SQL Injection is a critical vulnerability where attackers supply malicious input treated as part of a database query. By understanding how SQLi works, from concatenated user input to bypassing authentication, I've learned that the best prevention uses parameterized queries, input validation, and least privilege principles.

Security Database

OS-Level Vulnerability

Command injection allows attackers to execute arbitrary OS commands by supplying untrusted input to shell functions. I learned that preventing this requires avoiding shell invocation, using argument lists instead of string concatenation, validating input, and running processes with least privilege to minimize damage if exploited.

Security System

Client-Side Vulnerability

Reflected XSS occurs when user-supplied input is immediately reflected in a page's response without proper encoding. Through hands-on activities, I discovered how attackers inject malicious scripts to steal cookies, hijack sessions, or redirect users to phishing pages, and how output encoding prevents these attacks.

Security Web

Path Traversal Vulnerability

Local and Remote File Inclusion vulnerabilities allow attackers to include files from the server's filesystem or external sources. I learned that LFI can expose sensitive files through directory traversal (../../etc/passwd), while RFI enables remote code execution, both prevented by strict file whitelisting and input validation.

Understanding networking revealed how data travels across systems and where vulnerabilities exist. This hands-on experience demonstrated how encryption protects confidentiality, ensures authentication, maintains integrity, and provides non-repudiation in real-world digital communication.
